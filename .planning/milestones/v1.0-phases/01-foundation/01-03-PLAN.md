  <name>Task 2: Implement PWA start page redirect on dashboard</name>
  <files>src/routes/app/+page.svelte</files>
  <action>
In `src/routes/app/+page.svelte`, add a start page redirect that fires on mount:

1. In the `<script>` block, add:

   ```typescript
   import { goto } from '$app/navigation';
   ```

   (Already imported in current file)

2. Add a flag to prevent redirect from firing on subsequent navigations (only on initial PWA launch / direct dashboard visit):

   ```typescript
   let startPageChecked = $state(false);
   ```

3. In the existing `onMount` (or create one if the current code uses `$effect`), add the start page check BEFORE other data loading:

   ```typescript
   onMount(async () => {
   	// Check start page preference — redirect if needed (PWA launch)
   	if (!startPageChecked) {
   		startPageChecked = true;
   		try {
   			const res = await fetch('/api/preferences');
   			if (res.ok) {
   				const { preferences } = await res.json();
   				if (preferences.startPage === 'favorites') {
   					goto('/app/foods', { replaceState: true });
   					return; // Don't load dashboard data
   				}
   			}
   		} catch {
   			// Silently ignore — show dashboard as fallback
   		}
   	}
   	// ... existing dashboard data loading ...
   });
   ```

   Note: Per user decision, "PWA silently navigates to the chosen start page on launch -- no visual indicator". Using `replaceState: true` keeps browser history clean. The redirect goes to `/app/foods` since the favorites page is not built yet in this phase — update the redirect target to `/app/favorites` when that route exists in Phase 2.

4. To minimize flash of dashboard content before redirect (Pitfall 5 from research), consider adding a brief loading guard:
   ```typescript
   let ready = $state(false);
   ```
   Set `ready = true` after the start page check completes (either redirected or decided to stay on dashboard). Wrap the dashboard template in `{#if ready}...{/if}` so nothing renders during the preference check.

Important: The redirect target is `/app/foods` for now as a temporary stand-in for the favorites page (which does not exist yet). Add a code comment explaining this: `// TODO: Change to /app/favorites when favorites page is built (Phase 2)`.
