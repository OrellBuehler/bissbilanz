In `src/routes/api/auth/callback/+server.ts`:

1. Add import at the top:

   ```typescript
   import { extractLocaleFromHeader, isLocale } from '$lib/paraglide/runtime';
   ```

2. After the user creation/update block (after `[user] = await db...returning()` for both new and existing user paths), and before `const session = await createSession(...)`, add locale handling:

   For NEW users (inside the `if (!user)` block, after the insert):

   ```typescript
   // Detect browser language for new users
   const browserLocale = extractLocaleFromHeader(event.request);
   const detectedLocale = isLocale(browserLocale) ? browserLocale : 'en';
   // Save detected locale to user record
   await db.update(users).set({ locale: detectedLocale }).where(eq(users.id, user.id));
   user = { ...user, locale: detectedLocale };
   ```

   Note: The `locale` column must exist on the users table (added by Plan 01's migration). If Plan 01 executes in parallel, this code will work once the migration runs. The locale column has a default of 'en' so existing users get 'en' automatically.

3. After the session creation and session cookie setting, but before the redirect, set the PARAGLIDE_LOCALE cookie:

   ```typescript
   // Restore language preference
   cookies.set('PARAGLIDE_LOCALE', user.locale || 'en', {
   	path: '/',
   	maxAge: 34560000, // ~400 days, matches Paraglide's cookieMaxAge
   	httpOnly: false, // Paraglide reads this client-side
   	secure: config.infomaniak.redirectUri.startsWith('https'),
   	sameSite: 'lax'
   });
   ```

   Important: `httpOnly: false` because Paraglide's cookie strategy reads the cookie client-side. Use `maxAge: 34560000` to match Paraglide's default cookieMaxAge (avoids Pitfall 6 from research).

4. The `event` parameter must be destructured from the handler args. Update the handler signature if needed — the current signature is `({ url, cookies, getClientAddress })`. Add `request` to the destructuring:
   ```typescript
   export const GET: RequestHandler = async ({ url, cookies, getClientAddress, request }) => {
   ```
   The `request` object is needed for `extractLocaleFromHeader(request)`.

Note: The `user` variable in this file is typed from the select query. The `locale` property will be available after the schema migration adds it. TypeScript may need the schema to be updated first (Plan 01 Task 1).
</action>
