1. Update the `ChecklistItem` type in `SupplementChecklist.svelte` to include `timeOfDay` in the supplement object:
   `timeOfDay: string | null;`

2. Add a `$derived` that groups checklist items by `timeOfDay`. Use this ordering: 'morning', 'noon', 'evening', null (anytime). Each group has a label and items array. Only include groups that have items.

   ```
   const timeOrder = ['morning', 'noon', 'evening', null];
   const grouped = $derived.by(() => {
     const groups = new Map<string | null, ChecklistItem[]>();
     for (const item of checklist) {
       const key = item.supplement.timeOfDay ?? null;
       if (!groups.has(key)) groups.set(key, []);
       groups.get(key)!.push(item);
     }
     return timeOrder.filter(t => groups.has(t)).map(t => ({ timeOfDay: t, items: groups.get(t)! }));
   });
   ```

3. Update the template to render grouped sections. If there is only one group (e.g., all items are "anytime"), render items flat without a section header (preserves current appearance for users who haven't set timeOfDay). If there are multiple groups, render each group with a small header label using the i18n keys from Plan 01 (`m.supplements_time_morning()`, etc., `m.supplements_time_anytime()` for null).

   Group header: a small muted text label (like `<p class="text-xs text-muted-foreground font-medium pt-2 first:pt-0">`).
   Items within each group render the same as current (checkbox + name + dosage).

4. Add i18n key in en.json: `"supplements_history_missed": "Missed"` and `"supplements_history_adherence": "{taken}/{total} taken"`.
   Add corresponding German keys in de.json: `"supplements_history_missed": "Verpasst"` and `"supplements_history_adherence": "{taken}/{total} eingenommen"`.
