  <action>
    1. Add `weightEntries` table to schema.ts following the supplements pattern:
       - id: uuid primary key defaultRandom
       - userId: uuid notNull references users.id onDelete cascade
       - weightKg: real notNull (WGHT-02: kg only, no unit field)
       - entryDate: date notNull (user-local date, per P-W1 decision)
       - loggedAt: timestamp with timezone notNull (exact moment, WGHT-01)
       - notes: text (optional)
       - createdAt/updatedAt: timestamp with timezone defaultNow
       - Indexes: (userId, entryDate) and (userId, loggedAt)
       - No unique constraint on (userId, entryDate) — multiple entries per day allowed

    2. Run `bun run db:generate` to create migration. NEVER use db:push.

    3. Create validation/weight.ts with Zod schemas:
       - weightCreateSchema: { weightKg: z.coerce.number().positive().max(500), entryDate: z.string().regex(/^\d{4}-\d{2}-\d{2}$/), notes: z.string().optional().nullable() }
       - weightUpdateSchema: same fields but all optional (partial update)

    4. Create weight.ts server module following supplements.ts Result pattern:
       - createWeightEntry(userId, payload) — validate with weightCreateSchema, insert with loggedAt set to `new Date()`, return with .returning()
       - getWeightEntries(userId) — return all entries ordered by loggedAt DESC (for history list)
       - getWeightWithTrend(userId, from, to) — use raw SQL via db.execute(sql`...`) with CTE: DISTINCT ON (entry_date) to pick latest per day, then compute AVG window function for 7-day moving average (WGHT-05). Return entry_date, weight_kg, moving_avg ordered by entry_date ASC.
       - getLatestWeight(userId) — return single most recent entry (ORDER BY loggedAt DESC LIMIT 1)
       - updateWeightEntry(userId, id, payload) — validate with weightUpdateSchema, update WHERE id AND userId, return updated
       - deleteWeightEntry(userId, id) — delete WHERE id AND userId, return success boolean

  </action>
  <verify>
    Run `bun run dev` briefly to confirm migration applies cleanly (no startup errors). Check that `drizzle/` has a new migration SQL file for weight_entries table.
  </verify>
  <done>
    weightEntries table exists in schema with correct columns and indexes. Migration generated. Server module exports all 6 functions. Validation schemas export create and update schemas.
  </done>
