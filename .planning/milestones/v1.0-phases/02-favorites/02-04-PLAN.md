<task type="auto">
  <name>Fix imageUrl validation to accept relative paths</name>
  <files>src/lib/server/validation/foods.ts</files>
  <action>
Update the imageUrl field in foodCreateSchema (line 29) to accept both absolute URLs and relative paths starting with '/'.

Current validator:
```
imageUrl: z.string().url().optional().nullable()
```

Change to:
```
imageUrl: z
  .string()
  .refine((val) => val.startsWith('/') || val.match(/^https?:\/\//), {
    message: 'Image URL must be a relative path (starting with /) or absolute URL'
  })
  .optional()
  .nullable()
```

This allows:
- Absolute URLs: `https://example.com/image.webp` (for Open Food Facts imports)
- Relative paths: `/uploads/uuid.webp` (for local image uploads)

No other changes needed — foodUpdateSchema inherits from foodCreateSchema via `.partial()`.
  </action>
  <verify>
Run the TypeScript type checker: `bun run check`

Verify the change compiles without errors. No new type issues should appear.
  </verify>
  <done>
imageUrl validator accepts both absolute URLs (https://...) and relative paths (/uploads/...). The foodCreateSchema and foodUpdateSchema both reflect this change.
  </done>
