### Partial (2/31)

#### PREF-04: Widget reorder not applied on dashboard

**Phase:** 1 (Foundation)
**Issue:** The settings page allows drag-and-drop reordering of dashboard widgets and persists the order via PATCH /api/preferences. The dashboard fetches preferences and has `userPrefs.widgetOrder` in state. However, the three widget `{#if}` blocks in `src/routes/app/+page.svelte` (lines 236-244) are hardcoded in sequence: favorites → supplements → weight. The `widgetOrder` array is never iterated to control render order.

**Fix:** Iterate `userPrefs.widgetOrder` to dynamically render widgets in user-configured sequence.

#### PREF-05: favoriteTapAction not respected on dashboard widget

**Phase:** 2 (Favorites)
**Issue:** The standalone `/app/favorites` page reads `favoriteTapAction` preference and shows a `ServingsPicker` dialog when set to `'picker'`. The `FavoritesWidget` on the dashboard (line 108) always uses instant-log, ignoring the preference. Users who configured "choose servings" get inconsistent behavior.

**Fix:** Pass `favoriteTapAction` from `userPrefs` (already loaded) as a prop to `FavoritesWidget`, or have the widget read it directly.
