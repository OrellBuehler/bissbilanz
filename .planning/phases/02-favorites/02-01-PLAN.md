<task type="auto">
  <name>Task 2: Favorites server module and API route</name>
  <files>
    src/lib/server/favorites.ts
    src/routes/api/favorites/+server.ts
  </files>
  <action>
    1. Create `src/lib/server/favorites.ts` with two exported functions:

    **listFavoriteFoods(userId: string, limit?: number)**
    - SELECT from foods LEFT JOIN foodEntries ON foods.id = foodEntries.foodId
    - WHERE foods.userId = userId AND foods.isFavorite = true
    - GROUP BY foods.id
    - ORDER BY count(foodEntries.id) DESC (most-logged first)
    - LIMIT (default 50)
    - Return: array of `{ id, name, imageUrl, calories, protein, carbs, fat, fiber, type: 'food', logCount }`

    **listFavoriteRecipes(userId: string, limit?: number)**
    - This is more complex because recipes don't have direct macro columns.
    - Use a subquery or CTE approach:
      a. First, get favorite recipes: SELECT from recipes WHERE userId = userId AND isFavorite = true
      b. For each recipe, compute per-serving nutrition by joining recipeIngredients -> foods, summing macros, dividing by totalServings
      c. Also LEFT JOIN foodEntries ON recipes.id = foodEntries.recipeId to get log count
    - The query should use `sql` template literals for the computed nutrition columns:
      ```
      SUM(foods.calories * recipeIngredients.quantity / foods.servingSize) / recipes.totalServings
      ```
    - ORDER BY log count DESC, LIMIT
    - Return: array of `{ id, name, imageUrl, calories, protein, carbs, fat, type: 'recipe', logCount }`
    - Round calories to integer, macros to 1 decimal place

    2. Create `src/routes/api/favorites/+server.ts` with GET handler:
    - Extract userId via `requireAuth(locals)`
    - Accept optional `?type=foods|recipes` query param and `?limit=N`
    - If no type filter, return both: `{ foods: [...], recipes: [...] }`
    - If type=foods, return `{ foods: [...] }`
    - If type=recipes, return `{ recipes: [...] }`
    - Follow existing API error handling patterns (try/catch with handleApiError)
  </action>
  <verify>
    - `bun run check` passes
    - `curl http://localhost:5173/api/favorites` returns JSON with foods and recipes arrays (when authenticated)
  </verify>
  <done>GET /api/favorites returns favorite foods and recipes ranked by log count, with computed per-serving nutrition for recipes</done>
</task>

<task type="auto">
  <name>Task 3: Image upload API and file serving</name>
  <files>
    src/lib/server/images.ts
    src/routes/api/images/upload/+server.ts
    src/routes/uploads/[filename]/+server.ts
  </files>
  <action>
    1. Create `src/lib/server/images.ts`:
    - Export `processImage(file: File): Promise<string>`
    - Read the File into a Buffer via `Buffer.from(await file.arrayBuffer())`
    - Use sharp to resize: `.resize(400, 400, { fit: 'cover', withoutEnlargement: true }).webp({ quality: 80 }).toBuffer()`
    - Generate filename: `${crypto.randomUUID()}.webp`
    - Determine upload directory from `process.env.UPLOAD_DIR || './uploads'`
    - Create directory with `mkdir(dir, { recursive: true })`
    - Write the processed buffer to `${dir}/${filename}`
    - Return `/uploads/${filename}` (the URL path)
    - Export `UPLOAD_DIR` constant for reuse

    2. Create `src/routes/api/images/upload/+server.ts`:
    - POST handler accepting multipart form data
    - `requireAuth(locals)` for authentication
    - `request.formData()` to get the file
    - Validate: file must exist, must be image/* type, must be <= 10MB
    - Call `processImage(file)` to get the imageUrl
    - Return `{ imageUrl }` with status 201
    - Return 400 for invalid input, 401 for unauthenticated

    3. Create `src/routes/uploads/[filename]/+server.ts`:
    - GET handler to serve uploaded files
    - Read filename from params, validate it matches pattern `[a-f0-9-]+\.webp` (UUID format)
    - Read file from UPLOAD_DIR using `readFile`
    - Return with `Content-Type: image/webp` and `Cache-Control: public, max-age=31536000, immutable`
    - Return 404 if file doesn't exist
    - No auth required (images are public once uploaded — they have UUID filenames which are unguessable)

    4. Add `uploads/` to `.gitignore` if not already present.
  </action>

    - `curl http://localhost:5173/uploads/xxx.webp` returns the WebP image
  </verify>
